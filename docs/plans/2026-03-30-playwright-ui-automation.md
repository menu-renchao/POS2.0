# Playwright UI Automation Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Scaffold a maintainable Playwright + TypeScript UI automation project with a runnable smoke test for the POS frontend.

**Architecture:** Keep configuration at the root, tests under `tests/`, lean page objects under `pages/`, business flows under `flows/`, and shared setup under `fixtures/`, `test-data/`, and `utils/`. Treat employee switching as business context, not as a single global authenticated state.

**Tech Stack:** Node.js, TypeScript, Playwright Test

---

**Step 1: Write the failing smoke test contract**

Create `tests/smoke/home.smoke.spec.ts` referencing a `homePage` fixture and an `openHome` flow that do not exist yet.
