# Payment Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Add payment page and flow capabilities for cash and credit-card payment, plus minimal contract and live verification coverage.

**Architecture:** Introduce a standalone `PaymentPage` for shared payment UI structure and low-level actions. Keep only payment-entry hooks in `order-dishes` and `recall`, and put payment sequencing plus receipt-choice handling in `PaymentFlow`. Drive the work with a contract test first, then add the smallest live `To Go` payment checks.

**Tech Stack:** TypeScript, Playwright Test
